hexo+github搭建个人博客

  • github
  • hexo

前提：安装node.js和git并进行相应的环境配置（git版本不要太高）

##安装hexo
方法一：输入npm install -g hexo安装hexo

这种安装方法也是会因为被“墙”所以失效

方法二：使用淘宝NPM镜像cnpm install -g hexo-cli

输入hexo -v检查是否安装成功。

输入npm install hexo-deployer-git --save或者cnpm install hexo-deployer-git --save

##运行hexo
安装完成后，打开node.js的命令行，在你需要创建项目的文件夹下（如F:\hexo）输入hexo init，hexo 即会自动在目标文件夹建立网站所需要的所有文件。

##安装依赖包
输入npm install命令，这里也同样可能遇到被“墙”的问题，换输入cnpm install即可，（即在F:\hexo目录中安装 node_modules）

##开启服务器
输入hexo server

打开浏览器，输入localhost:4000,就可以在看到你的刚刚搭建的博客了，按Ctrl+C 停止Server。

输入hexo new "My New Post"新建一篇文章。会在F:\hexo\source_posts目录下生成一个markdown文件：My-New-Post.md

重新开启服务器就可以看到新建的文章。

注意：如果没有关闭服务器再新建文章，而是打开新的命令窗口直接新建文章，文章会被新建两次。

输入下面的命令，将markdown文件生成静态网页。hexo generate该命令执行完后，会在 F:\Hexo\public\ 目录下生成一系列html，css等文件。

##部署到Github
github上创建账号
去https://github.com/注册账号，并建立repository
建立仓库名必须是XXX.github.io

在F:\hexo文件下找到_config.yml文件，将里面的deployment项修改为：

deploy:

type: git

repository: https://github.com/username/usernme.github.io.git
branch: master

username为自己github的账号名。

##报错


输入npm install hexo-deployer-git --save或者cnpm install hexo-deployer-git --save我们应该在安装完hexo就执行此步。
再输入hexo generate

注意：有些新用户需要设置 ssh，否则上述命令会失败。ssh 的介绍和设置方法请看官方教程。https://help.github.com/articles/generating-ssh-keys/

每次修改完文件后要重新hexo g然后hexo d(这是都是命令hexo generate和hexo deploy简写，与上面的命令一致)
打开浏览器输入http://XXX.github.io 就是你的博客了。

报错spawn git ENOENT

Error: spawn git ENOENT

at Process.ChildProcess._handle.onexit (child_process.js:1053:32)
at child_process.js:1144:20)

未添加Git环境变量引起，添加Git与git管理库的环境变量即可；
D:\Git\bin;D:\Git\libexec\git-core

如果出现这样的错误，就是git的版本太高，我安装的是1.9.5的
